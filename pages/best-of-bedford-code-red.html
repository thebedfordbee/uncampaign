Best of Bedford: Code Red

Ten unbelievable things buried in the Bedford Town Code. Two answers are fake. One is real. Good luck.

What Is Code Red?

A Playful Peek Inside Local Government

The Bedford Town Code is 762 pages long. It covers everything from how bright a cabaret must be to how many swine require a Planning Board permit. It is, for the most part, not a lot of fun to read.

Code Red is a short trivia game built from that document. Each question gives you three options: two are invented, one is pulled directly from the Bedford Town Code. The question is whether you can tell the difference — and whether the real rule turns out to be stranger than anything we made up.

You do not need to know anything about municipal law to play. That's more or less the point. The Bedford Town Code has been sitting in public view for years, and most residents have never opened it. This is a small way to change that — one surprising sentence at a time.

How This Was Built

How This Was Built

This campaign is committed to showing its work. When AI helps build something useful, it should be explained and made visible — not hidden.

This page started with the 762-page Bedford Town Code. AI helped scan the document for odd, specific, and surprising provisions that might make residents stop and say, "Wait, that's real?" The questions were then reviewed, edited, and shaped into a short game so the code feels a little less like a PDF no one opens and a little more like something the community can actually explore.

Like the rest of The UNCAMPAIGN, the point is not to make technology look impressive. The point is to make local government easier to see.

Public Instruction Set 
Act as a civic game designer, municipal law researcher, and front-end builder tasked with creating a short trivia game built from the Bedford Town Code for a local civic campaign.

The tool to build is:
A Bedford Town Code trivia game that turns a 762-page municipal document into a playful civic literacy experience.

Context:
The Bedford Town Code is publicly available but almost never read by residents. It contains specific, surprising, and sometimes genuinely unbelievable provisions that most people would not expect to find in a local law. The goal of this game is to surface those provisions in a format that is approachable, entertaining, and honest — not mocking.

Design and maintain these priorities throughout the build:

1. Source Integrity
Every "real" answer must be pulled directly from the actual Bedford Town Code. No paraphrasing that changes legal meaning. Provide both the actual code language and a plain-English context explanation. Make the distinction between "real" and "invented" options as clear as possible without giving the answer away.

2. Tone
The tone should be: curious, civic-minded, lightly amused, never mean. These rules are real. The point is not to mock local government — it is to invite residents to actually look at it. A raised eyebrow is appropriate. Contempt is not.

3. Question Design
Each question should present one real provision and two invented options. The invented options should be plausible enough to be confusing but not so outlandish that they are obviously fake. The real answer should be surprising enough that it earns a reaction. Avoid options that are easy to guess through elimination.

4. Game Mechanics
One question at a time. Player selects an option and submits. Immediate right/wrong feedback. Reveal the actual code language and a short plain-English explanation for every question, regardless of outcome. Track score. Show final score. Allow leaderboard submission with initials and hamlet only — no email, no name.

5. AI Transparency
The page should clearly explain that AI was used to analyze the town code and identify surprising provisions. This is a feature, not something to hide. Show the instruction set. Make the process visible.

6. Design System Discipline
Match the visual language of the UNCAMPAIGN site. Same palette, spacing, card treatment, typography, and tone. This should feel like civic infrastructure, not a game show.

7. Leaderboard
Mirror the leaderboard architecture of the Bedford Typing Challenge. Use the same Google Apps Script endpoint with different action names. Seed with plausible community scores so the board is not empty at launch. Provide graceful fallback if the endpoint is unavailable.

Build output requirements:
- Complete single-file HTML
- 10 trivia questions from the Bedford Town Code
- Question-by-question submission flow
- Score tally and final score display
- Leaderboard with initials and hamlet
- Share functionality
- AI transparency section with instruction set reveal
- SEO meta tags including Open Graph and Twitter Card
- Accessible markup throughout
