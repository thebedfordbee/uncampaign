What is Best of Bedford?

Most campaigns spend money to get attention. This campaign is trying to earn it by building things people might actually find useful or interesting.

Best of Bedford is an ongoing series of small civic participation projects. The Bedford Typing Challenge is the first one: a 60-second typing game where Bedford residents compete to be the fastest typist in town, using passages about the actual history, geography, and civic life of the community.

It is not a gimmick. It is an experiment in what local civic participation can look like when it is more human, more participatory, and less dependent on money and pressure.

This campaign does not
  • Buy lawn signs — no sign saturation
  • Send mailers — no bulk mail campaigns
  • Run robocalls — no automated calls
  • Use paid ads — no purchased reach
  • Hire consultants — no political class
  • Pressure anyone — participation only

Why this exists

Local politics defaults to expensive habits. This campaign is trying something else.

The standard playbook for local campaigns is lawn signs, mailers, robocalls, and door knocking. It is expensive, repetitive, and most residents find it annoying rather than engaging. It also assumes the goal is to persuade people — which is not really the goal of an uncontested or lightly contested local race.

Bedford deserves a choice. Not because any particular candidate should win, but because uncontested local elections are genuinely bad for civic health. Participation matters more than persuasion here.

So instead of spending money and energy on tactics most people tolerate at best, this campaign is trying to build things: a working website, a civic Q&A tool, a meeting recap series, a community idea tracker, and now a small local typing competition. All of it built with free tools, modest AI assistance, and a $49.99 total budget cap.

The Bedford Typing Challenge is part of that experiment. It is civic participation without pressure, technology without pretense, and community without partisanship.

How this was built.

This page — the cover art, the typing game, the leaderboard, the passages, all of it — was designed with AI assistance using Claude Code. A human reviewed every passage for factual accuracy, every design decision for campaign tone, and every line of code for correctness and privacy.

AI did not write Don's positions. AI did not fabricate facts. AI helped build faster and cheaper — which is the point. This is a $49.99 campaign. Every dollar of savings is proof that low-cost local campaigns can still build quality tools.

Civic Typing Game — Claude Code Prompt

Build a self-contained browser typing speed game for a local community. Requirements: - 60-second countdown timer that starts on first keypress - Pool of 20–30 factual passages about your town or organization (90–110 words each, civic or educational content only — no persuasion text) - Live WPM, accuracy, and progress display using standard 5-chars-per-word WPM calculation - Character-level feedback: correct chars in green, incorrect in red, animated cursor at current position - Seeded leaderboard with ~70 realistic fake entries so the page is not empty at launch (WPM clustering around 34–52 WPM average, accuracy 91–99%) - Optional score submission: initials (2–3 chars) and neighborhood only — no email or personal data - Results card with shareable text snippet and copy-to-clipboard fallback - Mobile-friendly layout with a clear "best on desktop" message for touch devices - All CSS and JavaScript inline — no external dependencies except optional Google Font - Graceful fallback if any backend is unavailable Design: match your existing site's color palette and typography. Keep the interface polished and distraction-free. The game should feel serious, not like a rough demo. Backend (optional): If you have a Google Apps Script endpoint, submit scores with action="submitTypingScore" and {initials, hamlet, wpm, accuracy, passageId, roundDuration} and fetch the leaderboard with action="getTypingLeaderboard". Fall back to localStorage if the endpoint is unavailable.
